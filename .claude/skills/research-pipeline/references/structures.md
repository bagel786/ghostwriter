# Paper Structures by Venue

## arXiv Preprint (general)
Flexible. Use this when the user hasn't named a venue.
1. Abstract
2. Introduction
3. Related Work (or Background)
4. Methodology / Approach
5. Experiments / Evaluation (if empirical)
6. Results
7. Discussion
8. Conclusion
9. References

## NeurIPS / ICML / ICLR (ML conferences)
8-page limit (+ unlimited references). Double-blind review typical.
1. Abstract
2. Introduction (ends with contributions list)
3. Related Work
4. Background / Preliminaries (if needed — define notation, problem setting)
5. Method
6. Experiments
   - Experimental Setup (datasets, baselines, metrics)
   - Main Results
   - Ablation Studies
   - Analysis / Discussion
7. Conclusion
8. References
9. Appendix (supplementary — proofs, extra experiments, implementation details)

**Tone:** Precise and technical. Strong emphasis on empirical results and ablations.
Reviewers expect to see comparisons against strong baselines.

## CHI / CSCW / UIST (HCI conferences)
10-page limit (+ references). Single-blind or double-blind.
1. Abstract
2. Introduction
3. Related Work
   - [Thematic subsection 1]
   - [Thematic subsection 2]
4. Study Design / System Design
5. [Findings / Results / User Study Results]
6. Discussion
   - Implications for Design
   - Limitations
7. Conclusion
8. References

**Tone:** More discursive than ML papers. Related work sections are longer and more
synthetic. "Implications for design" is expected. Figures of the system/interface are
important.

## IEEE (INFOCOM, ICDCS, general IEEE conferences)
6-10 pages. Often single-blind.
1. Abstract
2. Introduction
3. Related Work (sometimes moved to end)
4. System Model / Problem Formulation
5. Proposed Approach / Algorithm
6. Performance Evaluation
   - Simulation / Experimental Setup
   - Results
7. Conclusion
8. References

**Tone:** Formal. Lots of math and system modeling. Results presented in tables and
figures with detailed parameter discussion.

## ACM SIGMOD / VLDB (database/systems)
12-page limit typical.
1. Abstract
2. Introduction
3. Background and Related Work
4. Problem Statement
5. System Design / Architecture
6. Implementation
7. Experimental Evaluation
   - Setup
   - Results
   - Scalability Analysis
8. Conclusion
9. References

## Nature / Science (high-prestige general science)
Word limit varies. Very different structure.
1. Abstract (structured: Background, Methods, Results, Conclusions — 150 words max)
2. Introduction (broad motivation → specific gap, ends with what this paper does)
3. Results (results-first, not methods-first — methods go at the end or supplementary)
4. Discussion
5. Methods (brief in main text; detail in supplementary)
6. References

**Tone:** Written for a broad scientific audience. Avoid jargon. Story matters as much
as results. Figures are primary — the paper tells its story through figures.

## Short / Position Paper (workshops, 4-6 pages)
1. Abstract
2. Introduction + Motivation (combined)
3. Approach / Position (shorter than a full methods section)
4. Preliminary Results or Expected Contributions
5. Conclusion and Future Work
6. References

## Survey / Literature Review Paper
1. Abstract
2. Introduction
   - Scope and methodology (how you selected papers)
3. Background
4. [Taxonomy section — organize the field into categories]
5. [Category 1]
6. [Category 2]
7. [Category 3] ...
8. Open Problems and Future Directions
9. Conclusion
10. References

**Tip:** The taxonomy is the hardest part and most important. Spend time designing the
right categories before writing.
